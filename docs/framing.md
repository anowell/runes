# Runes Framing

## Vision

Runes is a developer-first work management system defined by a return to simple, file-and-VCS-native workflows. It should be low ceremony for personal and shared work alike while allowing high-signal collaboration when it matters.

## Guiding Principles

- Reduce Friction: keep the CLI fast, the required inputs minimal, and inference strong so daily work happens with little thought about tooling.
- Blazingly Fast: instant feedback feels like a native tool; latency or waiting kills the low-friction goal.
- Embrace Developers: build for the terminal, editors, and version control; keep documents readable and fully scriptable.
- Simple, Familiar Mental Models: treat projects as folders, records as files, and everything as searchable, scriptable, portable artifacts on disk.
- Flexible: accommodate many workflows, stay extensible, and let teams define their layers without confusing the core experience.

Runes keeps issue history independent from code history while making it easy to link back to the code side when needed.

## Future Exploration (Beyond v1)

- Goal alignment and resource allocation
- Decision lifecycles, from lightweight notes up through RFC-like decisions
- Estimation with forecasting and reliability signals
- Advanced insights and reporting

These ideas should sit atop a low-friction, fast, flexible project-management foundation.

## v1 Success Criteria

- Fully capable CLI for day-to-day issue management
- Independent stores with robust move/sync flows
- Clean, stable on-disk schema
- Strong search/filter/list UX on local data
- No web UI dependency
